================================================================
  FIX: TAURI NSIS CONFIG ERROR
================================================================

The build failed because tauri.conf.json had:

  - "nsis" listed as a build target
  - A windows.nsis block with "installerHooks"

Tauri 1.6 doesn't support installerHooks (it was added in
Tauri 2.0). And we don't want NSIS at all - we ship MSI.

Both have been removed. The config now only builds MSI.

The orphaned desktop/src-tauri/installer/installer.nsi file
has also been deleted since nothing references it now.

================================================================
  HOW TO RE-UPLOAD
================================================================

1. Unzip somewhere temporary.

2. View -> Show -> Hidden items so .github folder is visible.

3. CRITICAL: also delete your local installer folder before
   replacing files (Windows won't auto-delete folders that
   exist locally but not in the zip):

     File Explorer -> nhl-scoreboard\desktop\src-tauri\installer
     Right-click -> Delete

4. Drag everything inside the unzipped folder onto your local
   nhl-scoreboard folder. Replace all when asked.

5. Verify desktop/src-tauri/tauri.conf.json:
   "targets" should only list ["msi"]
   No "nsis" anywhere
   No "installerHooks" anywhere

6. GitHub Desktop:
     Summary: "fix: remove nsis target + installerHooks (tauri 1.6)"
     Commit + Push

7. Wait ~10 min for the new .msi.
