================================================================
  LIVE TICKING CLOCK
================================================================

The NHL API source now polls api-web.nhle.com every 1 second
instead of every 10 seconds. The clock will tick second-by-second
in real time during live games.

Two changes:

  1. NHLSource.poll_interval_sec default lowered from 10.0 to 1.0
  2. Engine async sleep tightened from 0.25s to 0.1s so the
     1Hz polls actually fire on schedule

================================================================
  TRADEOFFS TO BE AWARE OF
================================================================

  - ~3,600 requests/hour against api-web.nhle.com per running
    client. The NHL API is public but doesn't publish a documented
    rate limit. If you ship this to many users running the app
    concurrently, you may get throttled or temporarily blocked.

  - The clock will appear to "lag" or "skip" by ~1 second
    occasionally because the API itself doesn't update perfectly
    in lockstep with real time. This is normal.

  - Existing project files saved before this update will still
    have their old poll_interval_sec value (10.0) in the JSON.
    To get the live tick on existing projects:
      Open the project -> Source panel ->
      change Poll interval to 1.0 and save.
    New projects use 1.0 by default.

  - You can change the interval per project at any time via the
    Source panel (the Pydantic model still allows any float).

================================================================
  HOW TO RE-UPLOAD
================================================================

1. Unzip somewhere temporary.
2. View -> Show -> Hidden items so .github folder is visible.
3. Drag everything inside the unzipped folder onto your local
   nhl-scoreboard folder. Replace all when asked.
4. GitHub Desktop:
     Summary: "live ticking clock - poll api every 1s"
     Commit + Push
5. Wait ~10 min for the new .msi.
